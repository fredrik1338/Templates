# Tech meeting notes

In this document we store the meeting notes for the weekly recurring tech meeting.

## Notes

### Protocol

1. Add actions points as topics are discovered (outside of meeting)
2. At start of meeting assign moderator
3. Goes through existing actions points, check status and either:
   1. Update action point
   2. Conclude action point
4. Go through Discussion points
   1. The team prioritizes the discussions points
   2. The writer of the point explains the context and what to discuss
   3. Openly discuss the point. The moderator is responsible for managing deviations
   4. Conclude discussion points
   5. Write action point if needed and assign it
5. Close meeting
   1. Check that all new action points have an assignee

### Action points

Action points should generally be possible to do quickly.
For larger tasks an AP should instead be to write features and/or stories to do the task.

E.g. Write a feature for implementing X, schedule meeting with person Y to answer the following questions...

### Discussion points

A discussion point should in general be phrased as a question.
Some context should be provided and if possible one or more desired outcomes of a discussion.

E.g. Component X of application cannot handle the increasing load.
We can proceed with Y or Z but not both. Which do we prefer?
